Problem Description

For this project, the position of the object is not known in advance. The goal is to program the robotic arm to take a sequence of pictures from different directions in order to find the exact position of the object. We have to develop a contingency plan of the arm trajectory and can process the images with any package available (e.g. opencv). The last computation has to be made/verified with images taken from the lowest possible position of the camera above the object.

Approach

To locate the object, some image processing is required. We used BoofCV for HSV color segmentation.

It is also necessary to plan the robotic arm to enable inspection of the entire table surface to search for the object. Rather than using a predetermined search pattern, we created a general algorithm that determines what position to take each image from, given the current belief state. This algorithm is described in more detail on the Algorithm page.

Results

The algorithm typically locates the first point in 1-3 images, and finds its exact position in another 1-2 images. It then finds the second point in 1-2 images, and narrows its position down in 1-2 images. It finds then third point in 1-2 images, and narrows its position down in 1-2 images. Not including verification images, the algorithm typically fully determines the object's position and orientation in 7-10 images. (And finds the exact position of the first point in 3-5 images.)
